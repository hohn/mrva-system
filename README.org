* Overview

MRVA (Multi-Repository Variant Analysis) System enables running CodeQL queries across multiple repositories simultaneously.

- *mrva-system* is the blueprint (composition, interfaces, procedures)
- *mrva-deploy* is the scratch paper (runtime state, secrets)

* Submodules

| Module         | Purpose                          | Language |
|----------------+----------------------------------+----------|
| mrva-go-hepc   | HTTP Endpoint Provider for CodeQL databases | Go |
| mrva-docker    | Container configs & Helm charts  | Docker   |
| mrvaserver     | MRVA REST API server             | Go       |
| mrvaagent      | CodeQL analysis worker           | Go       |
| mrvacommander  | Shared library (HEPC client, etc.)| Go      |
| gh-mrva        | GitHub CLI extension             | Go       |

* Docker Images

All custom images use the =codeql-mrva-= prefix:

| Image                    | Source                     |
|--------------------------+----------------------------|
| codeql-mrva-server       | containers/server          |
| codeql-mrva-agent        | containers/agent           |
| codeql-mrva-hepc         | mrva-go-hepc               |
| codeql-mrva-ghmrva       | containers/ghmrva          |
| codeql-mrva-vscode       | containers/vscode          |

Images are published to =ghcr.io/data-douser/=.

* Quick Start

#+BEGIN_SRC bash
# Clone with submodules
git clone --recursive git@github.com:data-douser/mrva-system.git
cd mrva-system

# Or initialize submodules after clone
git submodule update --init --recursive

# Deploy via Helm (requires Kubernetes cluster)
helm install mrva ./submodules/mrva-docker/k8s/codeql-mrva-chart \
  --namespace mrva --create-namespace \
  -f production-values.yaml
#+END_SRC
